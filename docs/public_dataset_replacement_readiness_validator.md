# Public Dataset Replacement Readiness Validator

## Version

`v0.4.0-a23`

## Purpose

This step validates whether public dataset replacement candidates generated by the `v0.4.0-a22` public dataset replacement workflow are ready for real-file replacement validation.

The validator does not download, overwrite, or modify real public data. It inspects the a22 replacement plan and replacement manifest inventory to determine whether each candidate is ready for downstream replacement validation.

## Inputs

Default request file:

```text
configs/public_data_sources/public_dataset_replacement_readiness_request.yaml
```

The request points to the a22 outputs:

```text
outputs/public_data_acquisition/multi_cancer_realdata_pilot/dataset_replacement_workflow/public_dataset_replacement_plan.tsv
outputs/public_data_acquisition/multi_cancer_realdata_pilot/dataset_replacement_workflow/replacement_manifest_inventory.tsv
outputs/public_data_acquisition/multi_cancer_realdata_pilot/dataset_replacement_workflow/public_dataset_replacement_summary.yaml
```

## Outputs

Default output directory:

```text
outputs/public_data_acquisition/multi_cancer_realdata_pilot/dataset_replacement_readiness
```

Generated files:

```text
public_dataset_replacement_readiness_table.tsv
public_dataset_replacement_readiness_source_artifact_index.tsv
public_dataset_replacement_readiness_summary.yaml
public_dataset_replacement_readiness_report.html
```

## Readiness statuses

The validator assigns one readiness status per replacement candidate:

```text
ready_for_replacement_validation
not_ready_missing_real_file
blocked_missing_manifest_stub
blocked_missing_required_metadata
```

## Expected result after a22

The a22 workflow produced four replacement candidates and four replacement manifest stubs, but zero ready real files. Therefore the expected initial readiness result is:

```text
replacement_candidate_count: 4
ready_for_replacement_validation_count: 0
not_ready_missing_real_file_count: 4
```

## Run

From the repository root:

```powershell
python -m core.data.validate_public_dataset_replacement_readiness
```


## Agent role

This stage acts as a readiness gate between the a22 replacement workflow builder and future real public-data replacement execution.

It ensures that real replacement files, replacement manifest stubs, and required metadata are present before any downstream replacement or modality processing is attempted.
